# プロジェクト固有スラッシュコマンド

このプロジェクト（booking-app-google）専用のスラッシュコマンド一覧です。

## 使用可能なコマンド

### `/rm` - READMEを読む（グローバル）

READMEとドキュメントを読んで内容を把握します。

**実行内容:**
- README.md を読み込み
- docs/google-calendar-api.md を読み込み
- プロジェクト概要、最新の変更履歴、開発時の注意事項を把握

---

### `/dp` - デプロイ

変更をGitHubにプッシュしてNetlifyで自動デプロイします。

**実行内容:**
1. `git add .` で全ての変更をステージング
2. `git commit` でコミット（日本語メッセージ + Claude Code署名）
3. `git push` でGitHubにプッシュ
4. Netlifyが自動デプロイを開始（2-3分）

**注意:**
- コミットメッセージは変更内容を簡潔に記載
- 必ず Claude Code署名を含める

**使用例:**
```
/dp
```

---

### `/upd` - README更新（グローバル）

README.mdに変更内容を追記します（新機能追加時のみ）。

**実行内容:**
- 「## 概要」セクション - 新機能を追加した場合のみ
- 「## 主な機能」セクション - 新機能を追加した場合のみ
- その他必要に応じて関連セクションを更新

**注意:**
- 変更履歴（CHANGELOG.md）は `/logupd` で更新
- バグ修正・リファクタリングの場合は不要

---

### `/logupd` - CHANGELOG更新

CHANGELOG.mdに変更履歴を追記します。

**実行内容:**
1. CHANGELOG.mdの最上部（「## 最新の変更履歴」の直後）に今日の日付で新しいセクションを追加
2. 変更内容をカテゴリで分類して記載:
   - **機能追加**: 新機能の追加
   - **バグ修正**: バグ修正
   - **UX改善**: ユーザー体験の改善
   - **リファクタリング**: コード構造の改善
   - **ドキュメント整備**: ドキュメント関連
   - **セキュリティ**: セキュリティ関連の変更
3. 各項目は箇条書きで簡潔に記載
4. 技術的な詳細も含める（変更したファイル名、関数名など）

**フォーマット例:**
```markdown
### 2026年3月5日
- **機能追加**: 予約元を一覧で識別できるように場所情報と色分けを追加
  - 通常リンク: 場所「X DM」、colorId '2'（セージ色・緑）
  - PersonA（LINE連携）: 場所「公式LINE（公認）」、colorId '11'（トマト色・赤）
```

**注意:**
- README.mdは更新しない
- 日付は今日の日付を使用

**使用例:**
```
/logupd
```

---

### `/dplogupd` - 一括実行

README更新、CHANGELOG更新、デプロイを一括で実行します。

**実行内容:**
1. `/upd` - README.mdに変更内容を追記（新機能追加時のみ）
2. `/logupd` - CHANGELOG.mdに変更履歴を追記
3. `/dp` - 変更をデプロイ（git add → commit → push）

**注意:**
- 新機能追加の場合のみREADME更新、それ以外はCHANGELOGのみ更新
- コミットメッセージには変更内容を簡潔に記載
- 必ず Claude Code署名を含める

**使用例:**
```
/dplogupd
```

---

## コマンドの使い方

チャット入力欄で `/コマンド名` と入力すると、コマンドが展開されて実行されます。

**例:**
```
/dp
```

---

## よくある使用パターン

### パターン1: 新機能追加時
```
/dplogupd
```
→ README、CHANGELOG更新してデプロイ

### パターン2: バグ修正・リファクタリング時
```
/logupd
/dp
```
→ CHANGELOG更新してデプロイ（READMEは更新不要）

### パターン3: 開発開始時
```
/rm
```
→ プロジェクトの現状を把握

---

## グローバルコマンドとの関係

`/rm` と `/upd` はグローバルコマンド（全プロジェクト共通）としても設定されています。
プロジェクト固有の設定がある場合は、プロジェクト固有の方が優先されます。
