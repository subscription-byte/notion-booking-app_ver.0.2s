# Google Apps Script 双方向同期セットアップ手順

## 概要

このスクリプトは、NotionとGoogleカレンダー間で予約データを双方向に同期します。

- **Notion → Google Calendar**: Notionに手動追加された予定をGoogleカレンダーにコピー
- **Google Calendar → Notion**: Googleカレンダーに追加された予定をNotionにコピー
- **重複防止**: 既に移行済みのCSVデータは除外
- **期間制限**: 今日以降の予定のみ同期

## セットアップ手順

### 1. Google Apps Scriptプロジェクトを作成

1. [Google Apps Script](https://script.google.com/) にアクセス
2. 「新しいプロジェクト」をクリック
3. プロジェクト名を「予約システム双方向同期」などに変更

### 2. スクリプトコードを追加

1. `gas-bidirectional-sync.js` の内容を全てコピー
2. GASエディタに貼り付け
3. Ctrl + S で保存

### 3. Advanced Calendar Serviceを有効化（推奨）

1. 左サイドバーの「サービス」(⊕アイコン)をクリック
2. 「Google Calendar API」を選択
3. 「追加」をクリック

※ これにより、extendedPropertiesなどの詳細情報も同期可能になります

### 4. スクリプトプロパティを設定

1. 左サイドバーの「プロジェクトの設定」(⚙️アイコン)をクリック
2. 「スクリプト プロパティ」セクションまでスクロール
3. 「スクリプト プロパティを追加」をクリックして以下を設定:

| プロパティ名 | 値 | 説明 |
|------------|-----|------|
| `NOTION_TOKEN` | `secret_xxx...` | NotionのIntegration Token |
| `NOTION_DATABASE_ID` | `1fa44ae2d2c780a5b27dc7aae5bae1aa` | NotionデータベースID |
| `GOOGLE_CALENDAR_ID` | 環境変数と同じ | GoogleカレンダーID |
| `MIGRATED_CSV_DATA` | CSVの内容全文 | 移行済みデータ（下記参照） |

#### Notion Tokenの取得方法

1. [Notion Integrations](https://www.notion.so/my-integrations) にアクセス
2. 既存のIntegrationを選択、または「新しいインテグレーション」を作成
3. 「Internal Integration Token」をコピー
4. Notionデータベースページで「・・・」→「コネクト」→作成したIntegrationを選択

#### CSV設定方法

1. 以下のファイルをテキストエディタで開く:

2. 全文をコピー（Ctrl + A → Ctrl + C）
3. スクリプトプロパティの `MIGRATED_CSV_DATA` の値として貼り付け

### 5. 設定確認

1. GASエディタ上部の関数選択で `checkConfiguration` を選択
2. 「実行」ボタンをクリック
3. 初回実行時は権限承認が必要（画面の指示に従う）
4. 「実行ログ」を確認し、すべて「OK」と表示されることを確認

### 6. トリガーを設定

1. 左サイドバーの「トリガー」(⏰アイコン)をクリック
2. 右下の「トリガーを追加」をクリック
3. 以下のように設定:
   - **実行する関数**: `syncBothDirections`
   - **デプロイ時に実行**: Head
   - **イベントのソース**: 時間主導型
   - **時間ベースのトリガー**: 分ベースのタイマー
   - **時間の間隔**: 5分おき
4. 「保存」をクリック

### 7. 初回手動実行（動作確認）

1. GASエディタ上部の関数選択で `syncBothDirections` を選択
2. 「実行」ボタンをクリック
3. 「実行ログ」を確認し、エラーがないことを確認

## 動作の確認

### 同期ログの確認

1. 左サイドバーの「実行数」をクリック
2. 各実行をクリックしてログを確認
3. 以下のような出力が確認できます:


### 実際の同期確認

1. **Notion → Google Calendar のテスト**:
   - Notionデータベースに新しい予定を手動追加
   - 5分待つ（トリガーが実行されるまで）
   - Googleカレンダーに同じ予定が追加されていることを確認

2. **Google Calendar → Notion のテスト**:
   - Googleカレンダーに新しい予定を手動追加
   - 5分待つ
   - Notionデータベースに同じ予定が追加されていることを確認

## トラブルシューティング

### エラー: "NOTION_TOKEN が設定されていません"

- スクリプトプロパティが正しく設定されているか確認
- プロパティ名のスペルミスがないか確認

### エラー: "Notion API Error: Unauthorized"

- Notion Tokenが正しいか確認
- NotionデータベースにIntegrationが接続されているか確認

### エラー: "Calendar not found"

- `GOOGLE_CALENDAR_ID` が正しいか確認
- カレンダーの共有設定でサービスアカウントに権限が付与されているか確認

### 同期されない

- トリガーが正しく設定されているか確認
- 実行ログでエラーが出ていないか確認
- 予定の日付が今日以降であるか確認（過去の予定は同期されません）
- 同じ名前+日付の組み合わせが既存の場合はスキップされます

### 重複が発生する

- CSVデータが正しくスクリプトプロパティに設定されているか確認
- ユニークキー（名前+日付）が一致しているか確認

## 制限事項

1. **同期タイミング**: 5分ごとの実行なので、最大5分のラグが発生します
2. **拡張プロパティ**: GASの標準APIでは一部制限があります（Advanced Calendar Service推奨）
3. **過去の予定**: 今日より前の予定は同期対象外です
4. **仮登録（2099年）**: 自動的にスキップされます
5. **システムイベント**: 「（LINE認証のみ）」などのシステム用イベントはスキップされます

## メンテナンス

### CSV更新

新しいデータを移行した場合は、CSVを再エクスポートして `MIGRATED_CSV_DATA` を更新してください。

### ログの確認

定期的に「実行数」から同期ログを確認し、エラーが発生していないか確認してください。

### トリガーの調整

同期頻度を変更したい場合は、トリガーの設定を変更してください:
- 高頻度: 1分おき（API制限に注意）
- 低頻度: 10分おき、15分おきなど

## 注意事項

- **データ損失の防止**: 本番運用前に必ずテスト環境で動作確認してください
- **API制限**: NotionとGoogleのAPI制限に注意してください
- **バックアップ**: 定期的にNotionとGoogleカレンダーのバックアップを取得してください
